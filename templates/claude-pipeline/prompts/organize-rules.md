# タスク: プロジェクトルールの整理

## 目的

プロジェクトのルール文書群を、1次ドキュメントおよび直前のステップの整理結果と
整合する状態にし、重複と矛盾をなくす。

## 対象

- docs/development/ 配下のルール・フロー文書（存在するもの）
- CLAUDE.md（参照整合性の確認のみ）

ルール文書が1つも存在せず、かつ1次ドキュメントに開発ルールに相当する内容がある場合は、
docs/development/DEVELOPMENT_RULES.md として新規作成する（1次ドキュメントの内容のみを根拠とする）。

## 作業内容

1. 対象ファイルと、今回整理された1次ドキュメントの内容を読む。
2. 以下を行う。
   - 複数ファイルに重複して書かれているルールを1箇所（SSOT）へ集約し、他はリンクにする
   - 各ファイルの冒頭に「このファイルが何のSSOTか」を1〜2行で明記する
   - 1次ドキュメントで決まった新しいルール・変更があれば反映する
   - 文書間の矛盾を解消する（既存の設計・ルールを優先する）

## 禁止事項

- 新しいルールの発明（1次ドキュメントに根拠のない変更）
- ルールの実質的な緩和・厳格化
- 矛盾の解消方法が判断できない場合は、変更せず該当箇所に `<!-- 要確認: 理由 -->` を残す

## コミット

変更がある場合のみ、次の形式で1コミットする。

```
docs: プロジェクトルールを整理
```
