# タスク: CLAUDE.mdの整理

## 目的

CLAUDE.md を「AIが常に読み込む必要がある最小限の内容」だけに保ち、
それ以外は適切な置き場所へ切り出す。CLAUDE.md の肥大化を防ぐ。
CLAUDE.md が存在しない場合は、整理済みドキュメントに基づいて新規作成する。

## 方針

CLAUDE.md に含めるもの（目安: 150行以内）:

- プロジェクトの一言説明
- 技術スタックの要点
- 絶対に守るべきルールの要約（コーディング規約・命名・禁止事項など）
- 詳細ドキュメント・スキルへの参照リンク

切り出し先の使い分け:

| 内容 | 切り出し先 |
| --- | --- |
| 常時は不要だが参照される詳細ルール・背景 | docs/development/ 配下 |
| 特定タスクの時だけ必要な手順（例: PR作成、レビュー手順） | .claude/skills/<name>/SKILL.md（スキル化） |
| 設計に関する記述 | docs/design/ 配下（既存ドキュメントへ統合） |

スキル化する場合は `.claude/skills/<name>/SKILL.md` を作成し、
frontmatter に `name` と `description`（いつ使うスキルかを明記）を書く。

## 作業内容

1. CLAUDE.md（あれば）、docs/ 配下、.claude/skills/ 配下を読む。
2. 上記方針に従って CLAUDE.md を作成または再構成する。
   - 新規作成の場合は、要件・設計ドキュメントとリポジトリの実態（言語・フレームワーク・
     ディレクトリ構成）から導けるルールのみ書く
3. 切り出した内容には CLAUDE.md から参照（相対パスのリンク）を残す。

## 禁止事項

- 情報の削除（必ずどこかへ移動する）
- 既存ルールの実質的な内容変更（言い換え・整理のみ可）
- ドキュメントに根拠のないルールの発明
- 直前のステップで整理されたドキュメント構成を壊すこと

## コミット

変更がある場合のみ、次の形式で1コミットする。

```
docs: CLAUDE.mdを整理
```
