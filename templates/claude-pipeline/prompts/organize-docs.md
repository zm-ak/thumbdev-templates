# タスク: ドキュメントフォルダ構成整理

## 目的

アプリから登録された1次ドキュメント（__DOCS_PATH__/ 配下）を、
リポジトリの正式なドキュメント構成へ整理する。

## 前提

- リポジトリに既存のドキュメント構成ルール（フォルダ構成を定めたドキュメント等）が
  ある場合はそれを正とする
- 無い場合は以下を標準構成として使う。
  - docs/requirements/ — 要件
  - docs/design/ — 設計
  - docs/development/ — 開発ルール・フロー
  - docs/notes/ — 分類できないメモ類

## 作業内容

1. __DOCS_PATH__/ 配下の各ファイルを読み、内容を分類する。
2. 分類に従って整理する。
   - 既存の正式ドキュメントと**重複・更新関係**にある場合:
     正式ドキュメント側へ内容を反映（追記・更新）し、
     取り込み済みの1次ドキュメントは docs/archive/ へ `git mv` する。
   - 既存ドキュメントに対応がない**新規の内容**の場合:
     適切なフォルダへ `git mv` し、必要なら分かりやすいファイル名に変える。
3. docs/README.md（索引）が存在する場合は更新する。ドキュメントが3件以上になる場合は
   索引が無ければ作成する。

## 禁止事項

- ファイルの削除（アーカイブへの移動で代替する）
- 1次ドキュメントの内容の過度な要約・改変（原文の意味を保つ）
- ソースコードの変更
- __DOCS_PATH__/ が存在しない・空の場合は何もせず終了する

## コミット

変更がある場合のみ、次の形式でコミットする（移動と内容反映を分けてよい）。

```
docs: 1次ドキュメントをフォルダ構成へ整理
```
