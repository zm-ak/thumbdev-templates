# タスク: リポジトリ説明の生成

## 目的

1次ドキュメント（__DOCS_PATH__/ 配下）と既存のドキュメント・READMEを読み、
このリポジトリが何であるかを説明する文章を最新化する。

## 作業内容

1. 以下を読む。
   - __DOCS_PATH__/ 配下のすべてのファイル（存在する場合）
   - docs/ 配下の要件・設計ドキュメント（存在する場合）
   - README.md（存在する場合）
2. `.github/repo-description.txt` を作成または更新する。
   - GitHubのリポジトリ説明（About欄）に設定する文章
   - 1次ドキュメントと同じ言語・1行・120文字以内
   - このファイルはパイプラインPRのマージ後、自動でリポジトリ説明へ反映される
3. README.md を更新する。
   - 存在しない場合はプロジェクト概要のみの簡潔なREADMEを新規作成する
   - 存在する場合は冒頭の概要部分をドキュメントと整合するよう更新する
   - README全体の大規模な書き換えは禁止

## 禁止事項

- docs/ 配下の変更
- ドキュメントに書かれていない機能・特徴の記載

## コミット

変更がある場合のみ、次の形式で1コミットする。

```
docs: リポジトリ説明を更新
```
